# ✅ Исправление ошибки 400 Bad Request для карточек

## Проблема
При загрузке страницы "Мои карточки" и "Карточки инфлюенсеров/рекламодателей" возникала ошибка:
```
BadRequestException: Validation failed (numeric string is expected)
```

## Причина
Frontend отправлял запросы к API **без обязательных числовых параметров** `page` и `limit`:
- ❌ `/api/influencer-cards?userId=xxx`
- ❌ `/api/advertiser-cards?isActive=true`

Backend ожидал получить числовые значения для пагинации, но получал `undefined`, что вызывало ошибку валидации в `ParseIntPipe`.

## Решение
Исправлены frontend сервисы для добавления дефолтных значений пагинации:

### 1. Метод `getMyCards()`
Теперь всегда отправляет параметры пагинации:
- ✅ `/api/influencer-cards?userId=xxx&page=1&limit=100`
- ✅ `/api/advertiser-cards?userId=xxx&page=1&limit=100`

### 2. Метод `getCards()`
Автоматически добавляет дефолтные значения если не указаны в фильтрах:
```typescript
const filtersWithDefaults = {
  page: 1,
  limit: 100,
  ...filters  // Пользовательские фильтры могут переопределить дефолты
};
```

## Измененные файлы
1. `src/modules/influencer-cards/services/influencerCardService.ts`
   - `getMyCards()` - добавлены параметры `page=1&limit=100`
   - `getCards()` - дефолтные значения пагинации

2. `src/modules/advertiser-cards/services/advertiserCardService.ts`
   - `getMyCards()` - добавлены параметры `page=1&limit=100`
   - `getCards()` - дефолтные значения пагинации

## Результат
Теперь все запросы к API карточек включают корректные параметры пагинации:
- ✅ Загрузка "Моих карточек" работает без ошибок
- ✅ Фильтрация карточек инфлюенсеров работает корректно
- ✅ Фильтрация карточек рекламодателей работает корректно
- ✅ Вкладка "Избранное" работает корректно
