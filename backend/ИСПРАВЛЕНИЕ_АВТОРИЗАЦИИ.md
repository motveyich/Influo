# Исправление ошибки "Invalid or expired token"

## Проблема
Эндпоинты логина и регистрации возвращали ошибку 401 "Invalid or expired token", потому что были защищены глобальным JWT guard без декоратора `@Public()`.

## Решение
Добавлен декоратор `@Public()` к публичным эндпоинтам авторизации:
- `POST /auth/signup` - Регистрация
- `POST /auth/login` - Вход
- `POST /auth/refresh` - Обновление токена

## Что нужно сделать

### Вариант 1: Развернуть через Vercel Dashboard (Рекомендуется)
1. Перейдите на https://vercel.com/dashboard
2. Найдите ваш backend проект
3. Нажмите на вкладку "Deployments"
4. Нажмите "Redeploy" на последнем деплое

### Вариант 2: Развернуть через командную строку
```bash
cd backend
vercel --prod
```

### Вариант 3: Автодеплой через Git
```bash
git add backend/src/modules/auth/auth.controller.ts
git commit -m "fix: add @Public decorator to auth endpoints"
git push
```
Vercel автоматически развернет изменения.

## Тестирование после развертывания

1. Подождите 1-2 минуты пока завершится деплой
2. Очистите кэш браузера (Ctrl+Shift+Delete)
3. Обновите страницу (Ctrl+F5)
4. Попробуйте войти с валидными учетными данными
5. Авторизация должна пройти успешно без ошибки "Invalid or expired token"

## Что было изменено

В файле `backend/src/modules/auth/auth.controller.ts`:
- Импортирован декоратор `@Public()`
- Добавлен декоратор `@Public()` к методам `signup`, `login` и `refresh`

## Почему это необходимо

Глобальный JWT guard в `app.module.ts` защищает все эндпоинты по умолчанию. Декоратор `@Public()` помечает конкретные эндпоинты как публично доступные, обходя требование JWT аутентификации. Это необходимо для:
- Регистрации (нельзя иметь токен до регистрации)
- Входа (нельзя иметь токен до входа)
- Обновления токена (используется refresh токен, а не access токен)
